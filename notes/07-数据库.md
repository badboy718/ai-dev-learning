## MySQL基础

### 库操作

创建数据库 `create database [if not exists] XXX`

切换|使用 数据库 `use XXX`

查看所有数据库 `show databases`

### 数据约束

主键约束 `primary key` 非空唯一

非空约束 `not null` 不能为空

唯一约束 `unique`不重复

默认约束 `default`修饰列对应的值提前设置默认值

外键约束 `foreign key` 保证数据的完整性:

限制从表插入数据,同时也限制主表删除数据

### 单表查询

```sql
select *|字段名|sum(),count(),avg(),max().min()

from 表

where 字段名    比较（><=）|逻辑（and or not)|范围between|模糊 like %(任意多个字符) _（任意一个字符）  数据值

group by 字段名，字段名2

order by asc(升序 默认)|desc(降序)

limit 3,3 (第2页内容，每页3条);
```

### 多表查询

```sql
#不要用 交叉连接 不要用 数据量太大
select * from 左表,右表;

#内连接1
select *
from 左表,右表
where 左表.字段名=右表.字段名
#内连接2
select *
from
	左表 inner join 右表 on 左表.字段名 = 右表.字段名

#外连接 以一个表为主
select *
from
	左表 left|right join 右表 on 左表.字段名 = 右表.字段名

```

### 哈希操作

```python
import redis
r = redis.Redis(
    host='localhost',
    port=6379,
    db=0,
    decode_responses=True
)
#设置一个哈希字段
r.hset('user001','name001','001')
r.hset('user002','name002','002')
#设置多个字段（redis-py 4.0+ 写法）
r.hset('user003', mapping={
    'name003':'003',
    'age003':'30',
    'class003':'cls003'
})

#获取一个字段 r.hget
id = r.hget('user001','name001')
print(id)
#获取多个字段 r.hmget
ids = r.hmget('user003',['name003','age003'])
print(ids)
#获取所有哈希字段 r.hgetall
ids = r.hgetall('user001')
print(ids)

#获取所有key
data_keys = r.hkeys('user003')
print(data_keys)
#获取所有value
data_values = r.hvals('user003')
print(data_values)

#删除指定字段
r.hdel('user003','name003','age003')
ids = r.hgetall('user003')
print('剩下：',ids)
```

### TTL

```python
import redis

r = redis.Redis(
    host='localhost',
    port=6379,
    db=0,
    decode_responses=True
)
#设置键值对
r.set('key', 'value')
#是否存在 exists
exists = r.exists('key')
print('是否存在',exists)
#设置生存时间 expire
r.expire('key', 10)
#查看剩余时长 ttl
ttl = r.ttl('key')
print('剩余',ttl)
#设置新键值对，带有生存时间
r.setex('key2',20,'value2')
#永久存在 persist
r.persist('key1')

#查找所有键
all_keys = r.keys('*')
print('所有：',all_keys)

#删除key1键
r.delete('key1')
#查看是否存在
print(r.exists('key1'))
```

### Redis缓存思想

核心思路：用Redis作为MySQL的"高速缓存层"，减少直接查询数据库的压力。

**为什么需要缓存？**

- MySQL是磁盘数据库，查询相对慢
- Redis是内存数据库，读写速度快（纳秒级 vs 毫秒级）
- 热点数据反复查询时，走Redis能大幅提升性能

**缓存查询流程（三步走）：**

```
用户请求查询数据
    ↓
① 先查Redis ——有？→ 直接返回（缓存命中，速度快）
    ↓ 没有
② 查MySQL ——有？→ 把结果写入Redis + 设置TTL → 返回
    ↓ 也没有
③ 返回空
```

**核心代码思路：**

```python
# 伪代码
if redis.exists(key):
    # 缓存命中，直接返回
    return redis.get(key)
else:
    # 缓存未命中，查MySQL
    data = mysql.query(sql)
    if data:
        # 写入缓存，设置过期时间（防止内存无限增长）
        redis.setex(key, 300, data)  # 300秒过期
    return data
```

**为什么一定要设置TTL？**

- 数据可能变化，TTL保证缓存不会永久存旧数据
- 防止Redis内存无限增长
- 热点数据自动刷新，冷数据自动淘汰
